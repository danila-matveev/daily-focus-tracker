# ADR Log

## [2025-01-02 16:00] ADR-001: Использование python-telegram-bot для Telegram бота

### Статус
Принято

### Контекст
Необходимо было выбрать библиотеку для разработки Telegram бота на Python. Рассматривались варианты: python-telegram-bot, aiogram, telebot.

### Решение
Выбрана библиотека `python-telegram-bot[job-queue]==21.3` для работы с Telegram Bot API.

### Обоснование
- Официальная и активно поддерживаемая библиотека
- Встроенная поддержка job-queue для автоматических напоминаний
- Хорошая документация
- Асинхронная поддержка
- Простота использования для новичков

### Последствия
**Положительные:**
- Легко реализовать автоматические напоминания
- Хорошая документация упрощает разработку
- Активное сообщество и поддержка

**Отрицательные:**
- Может быть избыточной для простых задач
- Требует понимания асинхронного программирования

### Альтернативы рассмотрены
- **aiogram**: Более современная, но менее стабильная
- **telebot**: Проще, но нет встроенной поддержки job-queue

## [2025-01-02 16:00] ADR-002: Использование Mini App вместо встроенных функций бота

### Статус
Принято

### Контекст
Для трекинга прогресса по 6 сферам нужен более сложный интерфейс, чем простые кнопки и команды Telegram.

### Решение
Использовать Telegram Mini App (веб-приложение) для расширенного функционала трекинга.

### Обоснование
- Позволяет создать полноценный веб-интерфейс
- Можно использовать HTML/CSS/JavaScript
- Хранить данные на стороне приложения
- Легко обновлять без изменения кода бота

### Последствия
**Положительные:**
- Гибкость в разработке интерфейса
- Возможность использовать современные веб-технологии
- Разделение логики бота и трекера

**Отрицательные:**
- Требует отдельного деплоя веб-приложения
- Дополнительная инфраструктура

### Альтернативы рассмотрены
- **Inline кнопки с callback**: Ограниченный функционал
- **Inline режим**: Не подходит для сложных форм

## [2025-01-02 16:00] ADR-003: Хранение конфигурации через переменные окружения

### Статус
Принято

### Контекст
Нужно безопасно хранить токены и конфиденциальные данные.

### Решение
Использовать переменные окружения для всех конфиденциальных данных (BOT_TOKEN, OWNER_ID, WEBAPP_URL).

### Обоснование
- Безопасность: токены не попадают в код
- Гибкость: легко менять настройки без изменения кода
- Стандартная практика для Python проектов

### Последствия
**Положительные:**
- Безопасность
- Легко настраивать для разных окружений

**Отрицательные:**
- Требует настройки переменных окружения при деплое

### Альтернативы рассмотрены
- **Конфигурационные файлы**: Менее безопасно
- **Хардкод в коде**: Небезопасно и негибко
